✨ 技术与职业方向的一些整理

また、人生の交差点にたどり着いた気がする

🎯 1. 职业方向

  • 🏡 地域：希望在关西工作生活。
  • 🏭 行业：倾向制造业，尤其是 IT / 嵌入式 / IoT 等相关岗位。
  • 🚫 回避：SIer、文理不问 SE 。
  • 🌍 外资：新卒难度高（英语要求、职位稀缺、经验不足），但中途转职可能性大。
  • 💻 Web系：在日本并非IT唯一圣经，制造业也能维持较好的WLB。

🔎 2. 对日本 IT 行业的观察

  • 📊 文科SE现象：日本 IT 行业中大量“文科生SE”，是教育与就业结构的产物。
  • 🎓 理工优势：制造业技术职位通常限定理工科。
  • ⏳ Web系局限：迭代快、加班多，但国际迁移性有限。
  • 🌐 制造业价值：嵌入式/制造业 IT 经验在外资企业，国际上更具迁移价值。

🛠️ 3. 自己的技术积累

  • 🔬 研究室：RAG + LLM 修复 REST API 誤用tool。
  • 📱 个人项目：Android 图书管理 App（Java + SQLite + Google Books API）。
  • ⚙️ 实验项目：FPGA 流水线 ALU，能计算素数个数。

这些经历覆盖了 软件工程 → 系统开发 → 硬件实验 的不同层面，是未来继续打磨的基础。

📌 4. 技术发展的优先级

  • 🖥️ C：嵌入式与控制软件的核心，必须扎实（指针、结构体、排序查找等）。
  • 🐍 Python：研究/数据分析/测试的利器，在制造业 IT 里是“辅助语言”。
  • 💡 C++：车载系统、模拟、模型驱动开发中逐渐重要，可作为中长期加分项。
  • ☁️ 未来延伸：云平台 & IoT（如 AWS 数据上传、可视化）。

🚀 5. 接下来的行动

  • ⏱️ 短期（半年）：Python 强化数据处理 & 研究支撑，积累可展示的小项目。
  • 📚 中期（半年后）：用 C 重写基础算法 & 数据结构，形成肌肉记忆。
  • 🌐 长期：逐步接触 Cloud / IoT 应用，结合研究室经验，迈向制造业 DX / IoT。

📝 结语

真正的就业选择还要看企业需求与面试机会，
时间和精力应该投向哪里。

「未来的方向并不是答案，而是不断校准的过程。」
