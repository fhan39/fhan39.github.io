大规模语言模型时代，我的思考

Created2025-07-26|Updated2025-07-26|AI思考编程与研究
|Post Views:

不会灵活运用AI的人，极有可能被它取代？。

起点：我在思考什么？

最近在为研究室课题的选择做准备工作，接触到了一些与大语言模型 和 RAG（检索增强生成）相关的研究课题。
并且再次陷入了一下的内耗中：

  • GPT，Deep seek这么强，我还需要刷算法题吗？
  • 我的代码感觉不如GPT hh（理所当然）
  • 这研究像是，我只需要调试和喂gpt东西就可以做

但逐步深入之后，我的答案发生了转变。

我的判断：不是GPT取代我，而是我必须学会如何更好的驾驭它

很多人害怕 AI 抢走自己的工作。
但就像汽车取代马车，电灯取代蜡烛，
先进的生产力工具不会让你失业，
只会淘汰那些拒绝学习和改变的人。

也许真正的威胁不是 GPT 本身，
而有可能是你身边那个比你还会用 GPT 的人。

与其一味“写代码”，不如开始“搭系统”

这时候，程序员的角色也要进化：

传统工程师 GPT时代工程师
自己写所有逻辑 构建AI参与的工作流
调 API、调试 设计 Prompt、控制上下文
写工具 搭系统，让AI写工具

我目前感兴趣的研究课题是关于“REST API 自动修复系统”：
GPT负责生成修复代码，而我则负责整个系统的搭建与驱动：

  • 自动生成伪文档；
  • 接入RAG系统；
  • 构建Prompt模板；
  • 自动验证修复是否成功；
  • 分析失败案例并调优。

研究给我带来的不仅是“知识”，而是“未来工作方式的演练”

GPT不是万能的，它只懂“怎么回答”，不懂“该回答什么”。
我做的事情，才是系统的核心逻辑：

  • 谁来定义任务？
  • 怎么组合模块？
  • 利用AI解决现实问题，而非做“AI研究”本身

这或许是LLM时代的工程师所需要具备的素质吧。

结语：光写代码可能真的不行！！！

记录于2025年7月26日，一次真实的认知转变。
