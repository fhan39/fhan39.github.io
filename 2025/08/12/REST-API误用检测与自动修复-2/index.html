REST API仕様逸脱の自動検出とLLMによる修正 - 論文概要

Created2025-08-12|Updated2025-08-12|技术笔记毕业研究
|Post Views:

🔍 背景

REST APIを利用するプログラムでは、仕様書と異なるパラメータ形式や呼び出し方（仕様逸脱）がしばしば発生します。
これにより、動作不良や予期しない挙動が生じますが、既存の自動修正手法は仕様逸脱特有の問題に十分対応できていません。

🎯 研究目的

API仕様とソースコードを自動で照合し、逸脱箇所を特定。
さらに、LLMを用いて修正版コードを生成することで、仕様逸脱バグを効率的かつ正確に修正することを目指します。

🛠 提案手法の流れ

  1. 仕様抽出

    • OpenAPIなどの仕様書からパラメータ名、型、必須性などを抽出。

  2. コード解析

    • GitHub等のソースコードをAST（抽象構文木）化し、API呼び出し部とパラメータ値を特定。

  3. 逸脱検出

    • 抽出した仕様とコードを照合し、仕様と異なる箇所を特定（逸脱点）。

  4. 自動修正

    • 逸脱点の位置や値を含むプロンプトを生成し、仕様情報とともにLLMへ入力。
    • LLMが仕様適合の修正版コードを出力。

🧪 実験概要

  • 対象API：SwitchBot API
  • 比較条件：
    • 逸脱点情報を含むプロンプト
    • 逸脱点情報を含まないプロンプト
  • 評価指標：修正成功率（仕様適合度）

📊 主な結果

  • 逸脱情報ありのプロンプトの方が修正成功率が高かった。
  • 提案手法は、REST APIの誤用を高精度で修正できる可能性を示した。

💡 まとめポイント

  • 仕様抽出 + コード解析 を自動化することで、バグ位置を高精度に特定可能。
  • LLMに修正対象を明示すると、修正精度が大きく向上。
  • 実験ではSwitchBot APIを対象に有効性を定量的に確認。
